# RL2LLM 概念对应关系

- **state** → **当前可见的文本上下文**
- **action** → 给定当前上下文，模型对下一个 token 的概率分布
- **trajectory** → 从 prompt 开始到整段 response 结束的整条生成过程（s₁, a₁, r₁, s₂, a₂, r₂, …）
- **reward** → 整段回答生成后给出的 sequence-level 奖励
- **environment transition** → 环境转移是确定性的文本拼接
- **policy** → 当前模型在每个位置上输出的 next-token 分布
- **value** → 当前生成前缀状态下，后续能获得的期望回报
- **advantage** → Aₜ = 实际回报 − baseline

## 问题 
1. 在 LLM 里，为什么说 action 是 token，而不是整段回答？ 
因为 LLM 是自回归生成模型，策略定义在每一步的下次token生成概率上。

2. 如果 reward 是整段回答结束后才给，那 PPO 怎么还能更新中间那些 token？
通过奖励，价值，GAE传回去。例子
abcd，这里奖励函数只给abcd这一句话一个奖励，假设是1，然后前面的没有奖励，都是0，但是可以得到每一时刻的回报，$G_t = \sum_{k=t}^T r_k$ ,结果是：
- 对 t=1：$G_1 = r_1+r_2+r_3+r_4 = 1$
- 对 t=2：$G_2 = r_2+r_3+r_4 = 1$
- 对 t=3：$G_3 = r_3+r_4 = 1$
- 对 t=4：$G_4 = r_4 = 1$
现在有了回报，然后根据价值就可以得到每一时刻的优势：**A_t = G_t - V(s_t)**
对于不同的位置有不同的价值，类似$V(s1​)=0.2,V(s2​)=0.5,V(s3​)=0.8,V(s4​)=0.9$，再通过GAE来得到广义估计优势，
$At_{GAE}​=δt​+γλδt+1​+γ2λ2δt+2​+⋯$ 
$δt​=rt​+γV(st+1​)−V(st​)$，
最后通过rt​(θ)往神经网络里面进行传递

3. 为什么 LLM 训练里经常需要 KL penalty？
如果只优化 reward，模型容易 reward hacking，能力退化。KL penalty 用来约束当前策略不要偏离 reference policy 太远。

4. 如果某个 token 的 advantage 为正，但 ratio 已经非常大了，PPO 会倾向怎么处理？
当 advantage 为正，且 ratio 已经超过 clip 上界时，PPO 的 surrogate objective 不再鼓励继续增大这个 token 的概率。

5. 如果 GRPO 主要改的是 advantage 的来源，那你猜它为什么会去看“同一个 prompt 下的一组回答”而不是只看单条回答？
一个回答绝对值到底该给多少分”很难稳定，但同一道题下，这几个回答谁更好”往往更容易判断。
**GRPO 不是主要想通过多答几次来估计一个更准的均值，而是想通过组内比较，直接构造谁高于组均值、谁低于组均值的训练信号。**
这就是它为什么能部分替代 critic，PPO需要具体的模型来给与奖励和价值，但是这里直接**在同一个 prompt 的条件下，哪些回答更值得提升概率，哪些更该压低。**


# GRPO 的核心直觉

PPO 的想法是**拿 critic 当参照物**
需要知道这条轨迹比“预期”好多少。  要一个 value/critic 来估计 baseline。
GRPO 的想法是**拿同题其他回答当参照物**
对同一个 prompt，干脆采样一组回答。  最好的那些，自然应该被鼓励；最差的那些，自然应该被压制。  直接利用组内相对表现，来构造 advantage-like 信号。

## 例子
prompt：证明 $\sqrt{2}$ 是无理数
 1. 回答 A:逻辑完整，证明正确  reward = 0.95

2.  回答 B:大体对，但中间有点跳步  reward = 0.70

3. 回答 C:有思路，但推导混乱  reward = 0.25

4.  回答 D:明显错误  reward = -0.20

PPO :- 想办法估一个 V(s)V(s), 再算每条轨迹上 token 的 advantage
GRPO:- 不急着学一个 critic,这 4 条回答已经足够形成一个排序,A、B 应该被鼓励,C、D 应该被压制,而且鼓励/压制强度，**由它们相对组均值的程度决定**

## 问题
1. 如果一个 prompt 很难，导致这一组回答的 reward 整体都偏低，为什么组内比较仍然有意义？
因为这是根据组内的整体情况来使得优势的更加优势，减少劣势的输出，这肯定是有意义的，还是会让模型往好的地方前进的，即使一个 prompt 很难，导致整组 reward 都偏低，**组内排序仍然保留了“谁相对更好”** 这个信息。  
GRPO 要利用的正是这个“同题下的相对优劣”，而不是跨题比较绝对分数。
2. 为什么说 GRPO 不只是“多采样几次取平均”，而是更像“构造相对优势信号”？
因为GRPO本质是通过组内的对比来使得模型往更好的方向前进，而不是取平均值来增强模型

# GRPO 的训练流程

**prompt → 采样一组回答 → 打分 → 组内相对化 → PPO式更新**

## 例子
prompt:2+2 等于几？
采样 4 个回答：
- A: “4” reward = 1.0
    
- B: “四” reward = 0.9
    
- C: “3” reward = -0.5
    
- D: “5” reward = -1.0
组均值：0.1
相对信号：
- A: 1.0−0.1=0.9
    
- B: 0.9−0.1=0.8
    
- C: −0.5−0.1=−0.6
    
- D: −1.0−0.1=−1.1
结果：
- A 和 B 都应该被鼓励
    
- C 和 D 都应该被压制
    
- D 压制最强
    
- A 鼓励最强
相对于PPO是 $A_t = G_t - V(s_t)$ ，而在 GRPO 里可以粗糙地理解成： $\text{relative signal} \approx \text{回答分数} - \text{组内基线}$

## 问题
1. 为什么 GRPO 里“组均值”或者“组内标准化”可以看成一种 baseline？
因为模型可以产生很多回答，有好有坏，但是这个是相对于模型来说他可以产生的一个平均程度，也就是一个基线成绩，组均值/组内标准化之所以像 baseline，是因为它提供了“在这个 prompt 下，这组回答的平均表现”作为参照。  某条回答不是单看绝对 reward，而是看它相对这个参照高还是低。
2. 如果一条回答 reward 很高，但它所在这一组的其他回答也都很高，那么它的相对更新强度会怎样？
可能不会很高，假设是0.9分，但是平均分可能有0.85分，那么强度也就0.05

# 公式

每条回答有 reward $r_i$。构造组内相对优势： $$ \hat{A}_i = \frac{r_i - \mu}{\sigma} $$ 其中： $$ \mu = \frac{1}{G}\sum_{i=1}^G r_i $$ $\sigma$ 是这组 reward 的标准差。 然后对回答 $o_i$ 中第 $t$ 个 token，定义概率比： $$ \rho_{i,t}(\theta) = \frac{\pi_\theta(o_{i,t} \mid q, o_{i,<t})} {\pi_{\text{old}}(o_{i,t} \mid q, o_{i,<t})} $$ 接着写 PPO 风格目标： $$ L_{\text{GRPO}} = \mathbb{E}_{i,t}\!\left[\, \min\!\left(\, \rho_{i,t}(\theta)\hat{A}_i,\; \mathrm{clip}\!\left(\rho_{i,t}(\theta),1-\epsilon,1+\epsilon\right)\hat{A}_i \,\right) \,\right] $$
$$ L = L_{\text{policy}} - \beta D_{\text{KL}}(\pi_\theta \parallel \pi_{\text{ref}}) $$



$\hat{A}_i = \frac{r_i - \mu}{\sigma}$
- $\hat{A}_i$：第 $i$ 个样本的**标准化优势值**
- $r_i$：第 $i$ 个回答的原始奖励/分数 
- $\mu$：当前批次内所有样本奖励的**均值**
- $\sigma$：当前批次内所有样本奖励的**标准差**
为什么要#减均值,除标准差
- 均值中心化：去掉本组整体水平，只看**相对好坏**
- 除以标准差：把波动缩到统一量级

这个公式可以看成**第 i 条回答在同组里的相对优势。**

PPO 优势函数 $(A_t)$
- 来源：**critic 网络 + GAE（广义优势估计）** 
- 形式： $[ A_t^{\text{PPO}} \approx Q(s_t,a_t) - V(s_t) ]$
- 本质：**动作价值 − 状态价值**，依赖价值网络拟合与时序折扣累积。 
GRPO 标准化优势 $(\hat{A}_i)$ 
- 来源：**同组内相对 reward 直接标准化** 
- 形式： $[ \hat{A}_i = \frac{r_i - \mu}{\sigma} ]$ 
- 本质：**样本奖励 − 组内均值**，再除以组内标准差做归一化，无 critic 网络。

$\rho_{i,t}(\theta)$
对于这条回答里的这个 token，新模型相对旧模型把它概率改了多少。这个和 PPO 是同一个东西，没变。

$L_{\text{GRPO}}$ 
就算这条回答相对更好，也不能把其中 token 的概率一下改太猛；  
就算这条回答相对更差，也不能一下压太狠。

## 问题
1. 为什么只用 $r_i−μ$ 还不够，很多实现还要再除以 σ？
因为如果不初上标准差会导致模型着重学习哪些得分高的内容，类似如果存在1，50，100分，那么差值就达到了50，然后另外一个是1，2，3，差值只有1，这个会导致模型过分学习到了前一个，而对下面一个没有平等学习。
2. 为什么说 GRPO 里的这个 $A^i$ 更像“回答级别 advantage”，然后再作用到 token 上？
GRPO 里的 $A^i$​ 先是对“第 i 条完整回答”算出来的相对优势信号，所以它天然是 **response-level** 的；然后训练时再把这个回答级别的信号作用到这条回答生成过程中对应的 token 上。

# 关键变量
## group size
**group size 决定了“组内 baseline”到底有多靠谱。**
**G 太小，组内参照系不稳定。**
**更大的组，通常意味着更可靠的组内排名和相对信号。但成本高**

## reward source
 GRPO 再厉害，本质上也还是根据 reward / verifier 给出来的信号去优化
这在数学、代码、规则任务里，通常更容易做得好，在开放式对话、创作任务里，往往更难。

## KL/reference policy
如果只追组内高 reward，模型可能会：

- 越来越偏向某些投机模式
    
- 输出更长、更模板化
    
- 学会骗 reward model
    
- 语言分布跑偏
## 长度偏置
 reward model、verifier 或人工偏好，天然更喜欢：

- 更长
    
- 更像标准答案模板
    
- 更“看起来认真”

这会导致学到的可能不是：
- 更正确的推理
    
而是：
- 更长、更像模板、更容易骗分的输出

## 问题
为什么说在 GRPO 里，group size 本质上是在影响“baseline 的可靠性”？

如果 group size 太小，这个 baseline 会非常依赖少数几条回答，容易受采样偶然性影响，不够稳定；  
如果 group size 更大，组内平均表现和波动尺度会更接近“这个 prompt 下模型真实的回答分布”，所以 baseline 更可靠。

如果 reward model 天然偏爱更长的回答，GRPO 可能会学出什么问题？
GRPO可能偏向说长的回答，但是意义可能并不如简洁的回答的效果
